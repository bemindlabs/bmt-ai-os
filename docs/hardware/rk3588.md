# RK3588 Setup (Orange Pi 5, Radxa ROCK 5B)

RK3588 boards offer the best price-to-performance ratio for ARM64 AI workloads. The 6 TOPS RKNN NPU accelerates smaller models; larger 7B models run on the high-performance CPU cores.

## Compatible Boards

| Board | RAM Options | Storage | Price |
|-------|-------------|---------|-------|
| Orange Pi 5 | 8, 16, 32 GB | eMMC + NVMe | ~$80–$150 |
| Orange Pi 5 Plus | 8, 16, 32 GB | eMMC + dual NVMe | ~$100–$170 |
| Radxa ROCK 5B | 8, 16, 32 GB | eMMC + NVMe | ~$120–$180 |
| Radxa ROCK 5B+ | 16, 32 GB | eMMC + NVMe | ~$140–$190 |

## Specifications (RK3588 SoC)

| Attribute | Value |
|-----------|-------|
| CPU | 4x Cortex-A76 + 4x Cortex-A55 |
| NPU | 6 TOPS RKNN (3 cores) |
| RAM | 8–32 GB LPDDR5 |
| Inference (1.1B, NPU) | ~18 tok/s |
| Inference (7B Q4, CPU) | 4–6 tok/s |
| Training (LoRA 1.5B, CPU) | ~3 hrs |
| Price | $100–$180 |

!!! note "NPU vs CPU inference"
    The 6 TOPS RKNN NPU is only effective for models up to ~1.5B parameters. Larger models (7B+) run on the A76 CPU cores, which deliver 4–6 tok/s — sufficient for interactive use.

## Prerequisites

- RK3588 board with 8GB+ RAM (16GB recommended)
- 64GB+ eMMC or NVMe SSD
- A suitable OS image (Armbian, Orange Pi OS, or Radxa OS)

## Installation

### 1. Flash the Board OS

=== "Orange Pi 5"
    Download Orange Pi OS (Debian-based) from [orangepi.org](http://www.orangepi.org/html/hardWare/computerAndMicrocontrollers/service-and-support/Orange-Pi-5.html).

    ```bash
    sudo dd if=orangepi5-debian-bookworm.img of=/dev/sdX bs=4M status=progress
    ```

=== "ROCK 5B"
    Download Armbian from [armbian.com](https://www.armbian.com/rock-5b/).

    ```bash
    sudo dd if=armbian-rock5b.img of=/dev/sdX bs=4M status=progress
    ```

### 2. Install Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
sudo systemctl enable --now docker
```

### 3. Deploy BMT AI OS

```bash
git clone https://github.com/bemindlabs/bmt-ai-os.git
cd bmt-ai-os
docker compose -f docker-compose.dev.yml up -d
```

### 4. Pull a Model

```bash
# Recommended for 8GB boards
docker exec bmt-ollama ollama pull qwen2.5-coder:7b

# Faster on NPU (1.5B class)
docker exec bmt-ollama ollama pull qwen2.5-coder:1.5b
```

## RKNN Configuration

RKNN acceleration requires the RKNN Toolkit2 and NPU kernel modules. These are pre-installed on Orange Pi OS and Armbian for RK3588.

To verify RKNN is available:

```bash
ls /dev/rknpu*
# Should show: /dev/rknpu  /dev/rknpu0  /dev/rknpu1  /dev/rknpu2
```

Pass the NPU device to containers:

```yaml
# In docker-compose.dev.yml, add under ollama service:
devices:
  - /dev/rknpu:/dev/rknpu
  - /dev/rknpu0:/dev/rknpu0
```

!!! warning
    RKNN model conversion is required. Standard GGUF/safetensors models do not run on the RKNN NPU without conversion via `rknn-toolkit2`. The built-in CPU fallback handles all models without any conversion.

## Performance Tips

- Use NVMe over eMMC for model loading — 5–10x faster cold starts
- On 8GB boards, avoid running multiple 7B model instances simultaneously
- Set `OLLAMA_KEEP_ALIVE=5m` to unload idle models and free RAM
- Pin the A76 cores to their maximum frequency for stable inference:
  ```bash
  echo performance | sudo tee /sys/devices/system/cpu/cpufreq/policy4/scaling_governor
  ```

## Known Issues

- RKNN driver is not mainlined in upstream Linux kernel — depends on vendor patches
- Some Armbian versions ship outdated NPU firmware; prefer Orange Pi OS for NPU workloads
- RK3588 runs warm under sustained inference; ensure adequate cooling (heatsink + fan)
